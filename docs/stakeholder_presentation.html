Slide 2

Why experience-study work benefits from a multi-agent copilot

The workflow is naturally segmented. Data prep, actuarial analysis, and visualization each have different rules, different failure modes, and different acceptable outputs.

  • A single general assistant tends to blur responsibility boundaries.
  • Specialist agents can be instructed narrowly and backed by explicit tool schemas.
  • The orchestrator can preserve user control by pausing between major stages.
Data Steward

Input trust

Profiles datasets, validates actuarial rules, engineers features, and writes the analysis-ready dataset.

Lead Actuary

Math trust

Runs deterministic dimensional sweeps and interprets A/E count and amount outputs cautiously.

Data Analyst

Communication trust

Generates chart artifacts from aggregated sweep outputs rather than from raw source data.

Orchestrator

Workflow trust

Routes intent and continuation prompts without silently chaining through the entire pipeline.

Test-backed behavior

The repo’s tests explicitly cover continuation routing, artifact creation, missing-feature guidance, and visualization dependency on fresh sweep outputs.

Slide 3

“Muscles and Brains” is the architecture, not just the tagline.

The point is to keep the system conversational without letting the model become the source of numerical truth.

Muscles: deterministic tools

Repeatable computation and artifact generation

  • tools/data_steward.py handles profiling, validation, and feature engineering.
  • tools/insight_engine.py performs A/E sweeps and Bayesian interval math.
  • tools/visualization.py writes standalone HTML reports.
  • Outputs land on disk as explicit handoff artifacts.
Brains: agents + orchestrator

Routing, tool selection, and explanation

  • Schema-bound tool calling is defined in agents/schemas.py.
  • Agents explain results, enforce role boundaries, and refuse inappropriate cross-role work.
  • The orchestrator classifies DATA_PREP, ANALYSIS, VISUALIZE, and CONTINUE.
  • The LLM remains a coordinator, not an actuarial calculator.
What changes with AI

Interface flexibility, step selection, and user-facing explanations.

What does not change

Validation rules, formulas, aggregation logic, and artifact formats.

Why it matters

Trust improves when the model can be useful without being authoritative about the math.

Slide 4

The workflow is staged, artifact-driven, and continuation-aware.

1
Orchestrator

Classify intent

Routes requests into general help, data prep, analysis, visualization, or continuation.

Key idea: do the first logical step, not the whole chain.

2
Data Steward

Prepare the analysis dataset

Profiles, validates, and engineers features into data/output/analysis_inforce.csv.

Input files remain immutable.

3
Lead Actuary

Run the deterministic sweep

Ranks cohorts and writes a run-specific CSV plus the latest alias sweep_summary.csv.

The agent interprets; the tool computes.

4
Data Analyst

Generate the browser report

Creates scatter or treemap HTML from the latest sweep artifact.

No fresh artifact, no fake success.

Continuation behavior

After data prep, continue queues the actuarial step. After analysis, continue queues visualization. This keeps the user in control while still preserving momentum.

Slide 5

Deterministic math is the trust boundary.

The repo’s actuarial credibility comes from where calculations live and from the rules that prevent the LLM from crossing that line.

MOC integrity

0 < MOC <= 1.0, and if MAC == 1, then MOC == 1.0.

Jeffreys prior

Credibility intervals use a Beta posterior with Beta(0.5, 0.5) prior parameters.

Amount fallback

Zero-claim cohorts fall back from MAF / MAC to MEF / MEC to keep amount bounds usable.

No LLM arithmetic

Agents are instructed to read tool outputs from disk and explain them, not derive them.

Interpretation model

The human-readable narrative can vary. The quantitative payload cannot.
